◎ 작성 시 유의사항 

‣ 연구책임자의 대표적 연구실적에 대한 요약문으로 각 실적별로 1페이지 이내로 기술하고, 증빙자료를 첨부(실적별로 서식을 복사하여 작성) 

‣ 대표적 연구실적은 논문, 특허, 기타실적을 합하여 연구책임자 5개 이내로 작성 

‣ 대표적 연구실적의 작성 최대 건수를 초과할 경우, 연구자별 작성 순 상위 5건만 인정 

‣ 최근 5년간(2021.1.1.이후)* 실적에 대해서만 작성 

 * 단, 성별에 관계 없이 연구자 본인의 출산휴가 및 육아휴직 기간이 2021.1.1. 이후에 포함되는 경우,  해당 기간만큼을 2021.1.1. 전으로 반영하여 실적 작성  

‣ 내용 작성 관련 설명 부분인“작성 요령(제출 시 삭제할 것)”은 삭제하고 작성 

‣ 작성표 내 셀 추가, 삭제 등 서식 변경 불가

[요약] 기존 클라우드 환경의 네트워크 성능 제어 기술은 대역폭 제한에만 초점을 맞추어, 실제 패킷 처리에 필수적인 CPU 자원 부족으로 인한 성능 저하 문제를 해결하지 못하는 구조적 한계가 있었다. 본 연구는 이러한 문제를 해결하기 위해 네트워크 트래픽 제어와 CPU 스케줄링을 유기적으로 연계하는 새로운 자원 관리 프레임워크인 Autothrottle을 제안하였다. 컨테이너의 네트워크 성능 목표(SLO) 달성 여부를 실시간으로 모니터링하고, 이에 필요한 CPU 할당량을 자율적으로 조절하는 피드백 제어 알고리즘을 리눅스 커널 레벨에서 독창적으로 구현함으로써, 대규모 트래픽 환경에서도 성능 변동성을 획기적으로 줄이고 목표 성능을 정밀하게 보장하는 성과를 거두었다. 이는 자원 간의 상호 의존성을 규명하고 통합 제어의 효율성을 입증함으로써 클라우드 시스템 소프트웨어 분야의 학문적 지평을 확장하는 데 기여하였다. 특히 발표 저널은 시스템 분야 대표 저널로서, IF 5.9 이상, JCR 10% 이내의 탁월한 연구 실적이다. 

 

[제안연구 연관성] 본 연구 실적은 제안 과제의 5차년도 핵심 목표인 “비용 효율적 하이브리드 추론 인프라”를 독자적으로 구축하기 위한 시스템 엔지니어링의 핵심 기술적 토대가 된다. 리눅스 커널 내부의 스케줄러를 직접 제어하여 시스템 전체의 성능 효율을 최적화한 경험은, 향후 고비용 GPU 자원에만 의존하지 않고 로컬 엣지와 클라우드 간에 트래픽과 연산 부하를 동적으로 분산 처리하는 "지능형 라우팅 및 오프로딩 기술"을 실현하는 데 필수적이다. 특히 대규모 워크로드 환경에서 검증된 커널 레벨 최적화 역량은, 수천 명의 학습자가 동시에 접속하는 AI 교육 플랫폼의 운영 비용을 절감하고 서비스 안정성을 확보하는 데 결정적인 역할을 수행할 것이다.
